## 커밋 스킬

변경사항을 분석해 의미 있는 단위로 나누고, 한국어 컨벤션에 맞는 커밋 메시지를 제안합니다.
사용자가 확인한 후 커밋을 실행합니다.

---

## 절차

### 1단계: 상태 파악

다음 명령을 실행해 현재 상태를 파악합니다.

```bash
git status
git diff HEAD          # 스테이지 여부와 무관하게 전체 변경사항
git log --oneline -5   # 최근 커밋 스타일 참고
```

아직 스테이지된 파일이 없어도 괜찮습니다. 이 스킬이 적절히 스테이징합니다.

### 2단계: 변경사항 분류

변경된 파일들을 **논리적 단위**로 묶습니다. 기준은 다음과 같습니다.

| 타입 | 기준 |
|------|------|
| `feat` | 새로운 기능, 컴포넌트 추가 |
| `fix` | 버그 수정, 오류 처리 |
| `refactor` | 동작 변경 없이 코드 구조 개선 |
| `style` | CSS, 디자인 변경 (로직 무관) |
| `chore` | 설정, 빌드, 패키지, 환경 변경 |
| `docs` | README, 주석 등 문서 변경 |

변경사항이 응집성이 있으면 **단일 커밋**으로,
서로 다른 관심사라면 **복수 커밋**으로 나눕니다.
지나치게 잘게 쪼개지 않고, 리뷰어가 한눈에 파악할 수 있는 단위로 묶습니다.

### 3단계: 커밋 메시지 작성

**형식:**
```
<타입>(<선택적 스코프>): <한국어 요약 (50자 이내)>
```

**예시:**
```
feat(header): TOON FORGE 브랜딩 및 별 장식 추가
style: Cuphead × Cartoon Network 테마 전체 적용
chore: Google Fonts 및 페이지 타이틀 변경
fix(card): 카드 호버 시 레이아웃 밀림 수정
refactor: CSS 변수 체계 정리 및 일관성 개선
```

**규칙:**
- 요약은 **동사 원형**으로 시작 (추가, 수정, 변경, 삭제, 개선)
- 구현 세부 사항보다 **무엇을/왜** 위주로 작성
- 마침표 없이 끝냄

### 4단계: 커밋 계획 제시

사용자에게 다음 형식으로 커밋 계획을 보여줍니다.

```
📋 커밋 계획

커밋 1/2  feat(auth): 사용자 인증 기능 추가
  • src/hooks/useAuth.ts (신규)
  • src/components/LoginForm.tsx (신규)

커밋 2/2  chore: 환경변수 설정 업데이트
  • .env.example (수정)

승인하시면 '네' 또는 '진행해줘'로 답해주세요.
수정이 필요하면 원하는 변경사항을 말씀해주세요.
```

### 5단계: 승인 후 커밋 실행

사용자가 승인하면:

1. 각 커밋 단위별로 해당 파일만 `git add`
2. `git commit -m "..."` 실행
3. 완료 후 결과 요약 출력

복수 커밋이면 순서대로 모두 실행합니다.

커밋 메시지 끝에 Co-Authored-By 라인을 추가합니다:
```
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

---

## 주의사항

- `.env`, `*.key`, `*secret*` 등 민감한 파일이 변경사항에 포함되어 있으면 **반드시 사용자에게 알리고** 해당 파일은 커밋에서 제외할지 확인합니다.
- `git add -A` 또는 `git add .`는 사용하지 않고, 파일을 명시적으로 지정합니다.
- 이미 커밋된 내용을 amend하지 않고 항상 새 커밋을 만듭니다.
- 사용자가 승인하기 전까지는 어떤 파일도 스테이지하거나 커밋하지 않습니다.
